# AI-smart-hardware
# AI 老年陪伴智能硬件 · 售前解决方案复盘

 **[在线预览 (Live Demo)](https://chemiui.github.io/AI-smart-hardware/)** 

##  项目简介
本项目包含业务售前复盘与前端工程实现两个维度的交付成果：
1. **业务层面：** 完整记录了百度智能云针对某老年康养智能硬件初创公司的 AI 赋能方案。拆解了如何通过“轻端重云”的多模态交互架构，解决低算力设备（ESP32）接入大模型的瓶颈，并利用“主动关怀”与“低延时 RTN”等特性切中客户的商业融资叙事，最终成功推动项目进入 POC 测试阶段。
2. **工程层面：** 本仓库包含了该复盘报告的**交互式前端展示单页 (SPA)** 源码。摒弃了重量级前端框架，纯手工打造了一个具有极客感、高保真且具备复杂图表交互能力的作品集网页。

##  业务解决方案架构 (Solution Architecture)
* **“轻端重云”底层逻辑：** 针对端侧极低的计算资源限制，设计了端侧仅保留高压缩率 SDK 与基础 3A 音频处理，将核心推理与多模态流转全部上云的协同架构，实现硬件 BOM 成本的零增加。
* **低延时通信网络 (RTN)：** 依托实时音视频传输通道，将端到端语音交互延迟压缩至 1.3s 级别，突破了老年群体对 AI 设备“迟钝感”的体验生死线。
* **情感化记忆与主动任务：** 规划并讲解了长短期记忆与主动唤醒引擎。将冷冰冰的“技术参数”转化为“设备能记住老人胸口痛并主动问候”的场景化表达，极大强化了客户产品的商业差异化壁垒。
* **商业模式锚定：** 以设备一次性 License 授权模式替换按量计费，为初创客户提供了可预测的成本结构，完美契合其融资计划书的财务测算需求。

##  前端技术栈与 UI 实现 (Frontend Tech Stack)
本项目前端展示页（`index.html`）完全由原生 Web 技术手写实现，展现了扎实的前端底层功底与极致的性能追求：

* **核心框架 (Core)：** HTML5, CSS3, Vanilla JavaScript (原生纯 JS)
* **设计规范与布局 (Design System)：**
  * **Bento Grid (便当盒网格)：** 采用 CSS Grid 与 Flexbox 混合布局，实现高密度信息的模块化与结构美感。
  * **原生明暗双主题 (Theme Toggle)：** 基于 CSS 自定义变量 (`--var`) 与本地存储 (`localStorage`)，实现无缝的 Light / Dark Mode 动态切换。
* **高阶数据可视化 (Advanced Visualizations)：**
  * **全手工 SVG 交互式架构图：** 未依赖任何图形库，直接使用原生 `<svg>` 标签、`<path>` 贝塞尔曲线与 `<defs>` 箭头标记，硬编码绘制了复杂的“端-网-云”三层架构数据流向图。
  * **动态坐标演算 Tooltip：** 结合原生 JavaScript 的 `getBoundingClientRect` 接口，动态计算鼠标悬停坐标，实现智能防溢出（自动判断上下方显示）的交互式悬停提示气泡。
* **微交互体验 (Micro-interactions)：**
  * 精心调优的 CSS `transition` 动画（如高光卡片的下划线延展、节点 Hover 发光效果）。
  * 原生 Intersection Observer API 实现的模块滚动进入视图时的平滑显现效果（Scroll Reveal）。

##  项目落地成果
* **商务推进闭环：** 成功推动客户完成企业账号注册与测试 License 发放，顺利启动核心能力的 POC（概念验证）测试，并挖掘出潜在的云资源迁移商机。
* **隐性知识显性化：** 深度复盘拜访实录，输出团队内部技术解读文章。将技术架构抽象为业务场景语言，形成高价值的组织级可复用资产。
